## Problem Statement

Current Thai voice dictation works but has two issues:
1. **Speed:** Must record entire audio before transcription starts (batch mode)
2. **Accuracy:** Small model struggles with difficult Thai words (tongue twisters, similar sounds)

User wants Wispr Flow-like experience: real-time transcription while speaking with high accuracy.

## What Makes This Cool

- **Real-time feedback:** Text appears AS you speak, not after
- **Better Thai accuracy:** Large-v3-turbo handles tongue twisters and similar-sounding words better
- **Still runs locally:** No cloud, no API costs, privacy-preserving
- **Apple Silicon optimized:** MLX makes large models fast on M1/M2/M3

## Constraints

- Must run on Apple Silicon Mac (M1/M2/M3)
- Must remain offline/local (no cloud APIs)
- Thai language focus
- Keep existing Thai post-processing (filler removal, segmentation, context modes)

## Premises

1. **Speed problem:** Streaming transcription will solve the "wait for results" issue
2. **Accuracy problem:** large-v3-turbo model will improve difficult word recognition
3. **Hardware is sufficient:** M1/M2/M3 Mac can handle larger models at acceptable speed
4. **Current post-processing works:** Keep thai_processor.py and thai_config.py

## Approaches Considered

### Approach A: Streaming Only
Add streaming transcription with current small model. Minimal changes, but accuracy doesn't improve.

### Approach B: Better Model + Streaming (CHOSEN)
Upgrade to large-v3-turbo + add real-time streaming. Best balance of effort and improvement.

### Approach C: Full Wispr Flow Clone
Complete rebuild with VAD, LLM post-processing, desktop app. Most features but highest effort.

## Recommended Approach

**Approach B: Better Model + Streaming**

### Implementation Steps

**Step 1: Add streaming support**
- Implement custom chunked transcription using MLX-Whisper's `transcribe()` with `clip_timestamps=[[0, 30]]` format (list of [start, end] tuples)
- Create ring buffer (3 chunks × 30s) for continuous audio capture (90s total buffer)
- Each transcription processes 30s windows with 5s overlap = 25s new content per inference
- **Overlap handling:** Use last 5s of previous chunk as prefix for context; merge transcripts by aligning on overlap region and keeping best prediction
- Handle partial results: print to stdout with `\r` carriage return for in-place updates
- Silence detection: 1.5s of audio below -40dB (energy threshold) to finalize current segment, not entire session
- **Memory budget:** ~2GB for 90s audio buffer + large-v3-turbo model (fits in 8GB+ RAM)

**Step 2: Upgrade model**
- Change model from `small` to `large-v3-turbo`
- First run will download ~1GB model
- Subsequent runs will use cached model

**Step 3: Integrate with existing pipeline**
- Keep `thai_processor.py` for post-processing
- Keep `thai_config.py` for customization
- `stream_thai.py` is the new default entry point (streaming mode)
- `transcribe.py` remains as legacy batch mode entry point
- Use `--batch` flag in `stream_thai.py` to fall back to batch mode if needed

**Prerequisite:** Verify `mlx-whisper>=0.4` supports chunked inference via `clip_timestamps` parameter.
